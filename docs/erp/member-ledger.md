# 会员域接口台账

## 1. 当前目标

会员域当前重点是：

- 区分会员明细接口和会员分析结果接口
- 确认搜索字段与分页是否会漏数
- 判断会员域是否已经存在可复用的主源接口

## 3. 关键过滤条件分析

### 3.1 日期条件

会员域的分析接口和中心接口不是同一种语义：

- `会员总和分析`、`会员消费排行` 强依赖销售时间窗口
- `会员中心` 当前样本更像状态型查询，不依赖时间窗口

这意味着：

- 会员中心适合作为会员主数据入口候选
- 分析接口更适合作为快照和对账视图

### 3.2 搜索与筛选

`会员中心` 当前已知字段：

- `condition`
- `searchval`
- `VolumeNumber`

当前已确认：

- `condition=''`、`searchval=''`、`VolumeNumber=''` 会返回当前账号可见范围内的会员集合，最新样本为 `1308` 行
- `searchval` 在 `condition=''` 时表现为全局过滤
  - 精确值可把结果收窄到 `1` 行
  - 宽松值如 `"1"` 可与 baseline 返回同一数据集
  - 无匹配值会返回 `0` 行
- `VolumeNumber` 会显著收窄会员集合，不是纯展示项
  - `1 -> 263`
  - `2 -> 94`
  - `10 -> 91`

当前仍需重点确认：

- `condition` 的合法取值集合
- `VolumeNumber` 的业务语义命名
- 是否存在服务端隐藏上限或分页

## 4. 维度重叠与失真风险

会员域当前的重叠比销售、库存少，但仍需区分：

- `会员中心` 更偏实体主数据
- `会员总和分析` 更偏结果视图
- `会员消费排行` 更偏排行结果视图

当前规则：

- 不把 `会员总和分析` 或 `会员消费排行` 当作会员事实主源
- `会员中心` 作为会员实体查询候选

---

## 5. 成本与金额字段现状

会员域当前更关注：

- 累计消费
- 储值
- 积分
- 消费频次

当前未把会员域视为成本字段探索主阵地。

但如果会员分析页出现：

- 客单毛利
- 会员毛利贡献
- 成本相关衍生字段

需要单独回记到成本专题文档。

---

## 6. 当前建议

- 主源候选：`会员中心`
- 快照候选：`会员总和分析`、`会员消费排行`

---

## 7. 后续待验证事项

1. `会员中心.condition` 的合法值集合
2. `会员中心` 是否存在隐藏分页或服务端上限
3. `VolumeNumber` 的业务语义命名与正式抓取是否需要纳入
4. `会员总和分析.type/tag` 是否是视图切换还是结果筛选
5. 不同角色账号下会员字段是否存在裁剪差异
