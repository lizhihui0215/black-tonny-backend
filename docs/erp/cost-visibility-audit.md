# 成本字段可见性审计

## 1. 目标

这份文档只回答一件事：

在当前合法账号与角色范围内，Yeusoft ERP 是否通过 API 返回成本相关字段，以及这些字段的返回状态是什么。

这里不讨论未授权访问，也不把“绕权拿成本”纳入计划。

研究目标限定为：

- 字段是否存在
- 字段是否返回列头
- 字段是否返回值
- 不同角色/菜单/接口是否有差异

---

## 2. 当前已确认事实

### 2.1 `销售清单`

真实样本文件：

- `tmp/capture-samples/raw/20260321-205200/销售清单.json`

当前已确认：

- `ColumnsList` 中存在：
  - `吊牌价`
  - `吊牌金额`
  - `成本价`
- `Data` 中：
  - `吊牌价` 有值
  - `吊牌金额` 有值
  - `成本价` 在当前样本下全为空

已核对事实：

- 总行数：`9839`
- `成本价` 非空行数：`0`

当前结论：

- `成本价` 不是字段缺失
- 也不是我们还没找到正确列名
- 更像账号权限、角色裁剪、菜单配置或字段可见性控制

### 2.2 `库存明细统计`

真实样本文件：

- `tmp/capture-samples/raw/20260321-205142/库存明细统计.json`

当前已确认：

- 存在 `RetailPrice`
- 当前未确认存在 `CostPrice`

当前结论：

- 该接口可以作为零售价观测来源之一
- 不能据此推断已经拿到成本字段

### 2.3 `库存多维分析`

当前已确认存在：

- `AStock`
- `AMoney`

但当前不能直接把 `AMoney` 解释成成本金额。

在未确认页面列头与角色差异前，`AMoney` 只能记为“金额聚合字段待解释”。

---

## 3. 当前分类规则

后续所有成本相关字段，统一按以下三类记录：

### 3.1 字段缺失

接口样本中完全没有该字段或列头。

例如：

- 当前 `库存明细统计` 样本未见 `CostPrice`

### 3.2 字段存在但为空

接口样本中有列头或字段名，但当前数据行值全部为空。

例如：

- `销售清单.成本价`

### 3.3 字段存在且可能受角色影响

当前样本不足以证明字段永远为空，需要通过不同角色或不同菜单权限进一步验证。

`销售清单.成本价` 当前应归入这一类。

---

## 4. 角色与权限审计方向

后续对成本字段的合法审计，统一按以下顺序推进：

1. 同一接口，不同角色账号对比
2. 同一角色，不同菜单权限集对比
3. 同一菜单，不同门店/组织范围对比
4. 同一业务域，不同接口对比

重点不是“能不能硬拿到”，而是：

- 哪类角色本来就应该看到成本
- 哪类角色只看到吊牌价/零售价
- 哪些接口在角色不同的情况下会裁剪字段

## 6. 当前结论

在当前样本基础上，更合理的判断是：

- 成本字段不是完全不存在
- 但当前账号下至少部分接口不返回具体成本值
- 下一步最有价值的是做合法角色差异审计，而不是继续盲猜字段名
